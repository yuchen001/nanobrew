TITLE: Show HN: Nanobrew - a 1.2 MB Homebrew replacement in Zig (10-500x faster)

URL: https://justrach.com/blog/building-nanobrew

TEXT (if submitting as text post instead of link):

I built nanobrew, a package manager that replaces Homebrew with a 1.2 MB Zig binary. Zero dependencies.

`brew install tree` takes 5.5 seconds for a 60KB binary. nanobrew does it in 10ms. That's 500x.

Some numbers (Apple Silicon, CI verified):

  tree (0 deps):    10ms warm, 0.68s cold   (Homebrew: 5.5s)
  ffmpeg (11 deps): 0.56s warm, 2.1s cold   (Homebrew: 19.6s)
  wget (6 deps):    33ms warm, 3.1s cold     (Homebrew: 5.8s)

How? No Ruby. No curl subprocesses. No otool. Everything is native Zig and runs in parallel. Streaming SHA256 during download instead of reading the file twice. APFS clonefile for zero-copy from a content-addressable store. Warm installs just create a reference to data already on disk. 3.5ms and done.

It's not just fast though. It does everything Homebrew does: bottles, source builds (cmake, autotools, meson), casks (.dmg/.pkg/.zip), third-party taps that parse Ruby formulas directly from GitHub, upgrade, rollback, pin/unpin, doctor, cleanup, bundle export, services. On Linux it handles .deb packages 2.8x faster than apt-get in Docker.

The tap support is something no other fast brew alternative has. `nb install user/tap/formula` fetches the .rb from GitHub, parses it line by line, resolves deps, installs. About 2 seconds.

Install: curl -fsSL https://nanobrew.trilok.ai/install | bash

GitHub: https://github.com/justrach/nanobrew

It's experimental but works well for common packages. Would love feedback on what breaks.
